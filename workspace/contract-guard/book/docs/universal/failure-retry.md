# 05 · 失败与重试契约（failure-retry）

> **适用领域**：`universal` `distributed` `ml-serving` `recommendation` `ecommerce` `realtime-iot` `multi-tenancy`
> **适用场景**：任何调用外部依赖、可能超时 / 报错 / 不可用的链路。
> **原则**：只否决"失败被吞 / 重试放大故障 / 无兜底雪崩"，不绑定熔断器实现。

## 一句话理解

失败要能被看见、能分类；重试不能没完没了，更不能把下游打垮。

## 契约速览

| 契约 | 不变式（红线） |
| --- | --- |
| **FR-1** 失败可见且可分类 | 任何失败都被记录并归类（可重试 / 不可重试） |
| **FR-2** 重试安全且有界 | 重试次数有上界，且只对可重试错误重试 |
| **FR-3** 依赖故障可降级不雪崩 | 单一依赖故障不导致整体不可用 |

## 契约 FR-1：失败可见且可分类

- **前置条件**：存在可能失败的操作（外部调用、IO、计算）。
- **后置条件**：任何失败都被捕获、记录（含上下文），并分类为"可重试"或"不可重试"，不被静默吞掉。
- **不变式**：不存在被吞掉的异常；每个失败都有可追溯的日志 / 指标。
- **验收标准**：注入各类失败 → 日志 / 监控均能看到，且分类正确。

## 契约 FR-2：重试安全且有界

- **前置条件**：对可重试错误启用了重试。
- **后置条件**：重试有最大次数上界、有退避（指数退避 + 抖动），且仅对幂等 / 可重试操作重试。
- **不变式**：重试次数有上界；不对"不可重试错误"（如参数非法）盲目重试。
- **验收标准**：
  - 持续失败 → 在上界次数后停止并上报，不无限重试。
  - 重试的写操作 → 满足 [幂等 IDEM-1](/universal/idempotency)，不产生重复副作用。

## 契约 FR-3：依赖故障可降级、不雪崩

- **前置条件**：系统依赖 ≥1 个可能不可用的下游。
- **后置条件**：单一依赖故障时，通过熔断 / 超时 / 兜底降级，保证核心路径仍可用或优雅失败。
- **不变式**：单一非核心依赖故障不导致整体服务不可用；线程 / 连接不被慢依赖耗尽。
- **验收标准**：摘除某下游 → 主流程返回兜底结果或明确错误，不超时堆积、不级联崩溃。

## 现场提问骨架

- "依赖超时怎么区分可重试和不可重试？重试有上界和退避吗？"
- "某个下游挂了，主流程是降级兜底还是一起挂？"

## 关联契约

- FR-2 的重试安全依赖 [幂等 IDEM-1](/universal/idempotency)；FR-3 的资源耗尽防护与 [资源生命周期 RES](/universal/resource-lifecycle) 相关。
