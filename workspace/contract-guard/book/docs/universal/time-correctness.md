# 07 · 时间与时区正确性契约（time-correctness）

> **适用领域**：`universal` `distributed` `engineering` `data-pipeline` `realtime-iot` `ecommerce`
> **适用场景**：任何涉及时间存储 / 展示 / 计算、定时任务、有效期、跨时区的逻辑。
> **原则**：只否决"时区算错 / 时钟回拨致错 / 边界差一"，不绑定时间库。

## 一句话理解

时间要分清"在哪个时区""用哪种钟"——存储统一用 UTC，测时长用单调钟，别被夏令时和时钟回拨坑了。

## 契约速览

| 契约 | 不变式（红线） |
| --- | --- |
| **TIME-1** 时间语义明确、存储统一 | 持久化时间带明确时区基准（统一 UTC） |
| **TIME-2** 时长测量用单调时钟 | 测量耗时 / 超时不受墙上时钟回拨影响 |
| **TIME-3** 边界时刻与差一正确 | 区间端点开闭明确，闰年 / 月末 / DST 不出错 |

## 契约 TIME-1：时间语义明确、存储统一

- **前置条件**：系统存储 / 传输 / 展示时间，且可能跨时区。
- **后置条件**：持久化与传输统一用 UTC（或带明确偏移的 ISO-8601），仅在展示层按用户时区转换。
- **不变式**：任一持久化时间值都带明确的时区基准，不存在"裸本地时间"歧义。
- **验收标准**：跨时区用户读写同一时刻 → 看到各自正确本地时间，底层值一致。

## 契约 TIME-2：时长测量使用单调时钟

- **前置条件**：代码测量耗时 / 实现超时 / 计算时间差。
- **后置条件**：耗时与超时基于单调时钟（monotonic），不受 NTP 校正 / 时钟回拨影响。
- **不变式**：测量的时间差恒 ≥0，不会因墙上时钟回拨出现负值或超时误判。
- **验收标准**：注入时钟回拨 → 超时判断与耗时统计仍正确，无负耗时。

## 契约 TIME-3：边界时刻与差一问题正确

- **前置条件**：存在时间区间、有效期、定时调度、按日 / 月聚合。
- **后置条件**：区间端点开闭明确；闰年 / 闰日 / 月末 / 夏令时切换 / 跨年等边界正确。
- **不变式**：区间端点开闭定义一致，不出现差一（off-by-one）漏算 / 重算。
- **验收标准**：
  - 有效期到期那一刻 → 行为与定义一致（含 / 不含端点明确）。
  - DST 切换日 / 闰日的定时任务 → 不漏跑、不重跑。

## 现场提问骨架

- "时间存的是 UTC 还是本地时间？展示时按谁的时区转？"
- "测耗时 / 超时用的是墙上时钟吗？时钟回拨会不会算出负数？"
- "有效期到期那一刻是含还是不含？闰日 / 夏令时的定时任务测过吗？"
