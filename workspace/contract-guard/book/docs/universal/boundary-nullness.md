# 04 · 边界与空值契约（boundary-nullness）

> **适用领域**：`universal` `engineering`
> **适用场景**：所有接收外部输入、处理集合 / 数值 / 字符串的逻辑。
> **原则**：只否决"未定义输入导致崩溃 / 脏数据落库"，不绑定校验框架。

## 一句话理解

脏数据和非法输入必须在进入业务核心之前就被挡在门外，绝不让它落库、也绝不让它参与运算。

## 契约速览

| 契约 | 不变式（红线） |
| --- | --- |
| **BN-1** 外部输入使用前必须校验 | 进入核心层的数据已满足类型与取值域约束 |
| **BN-2** 空集合 / 边界值有确定行为 | 分页 / 聚合 / 除法等对边界输入有定义 |

## 契约 BN-1：外部输入在使用前必须被校验

- **前置条件**：存在来自用户 / 上游 / 外部系统的输入（参数、报文、文件、回调）。
- **后置条件**：非法输入（缺失必填 / 类型错误 / 越界 / 超长 / 格式非法）在进入核心逻辑前被拒绝，返回明确错误，绝不落库、绝不参与运算。
- **不变式**：进入业务核心层的数据已满足声明的类型与取值域约束。
- **验收标准**：
  - 必填字段为 null / 空串 / 缺失 → 返回 400，不写库。
  - 数值越界（负数金额、超长字符串）→ 被拒绝并说明原因。

## 契约 BN-2：空集合 / 边界值有确定行为

- **前置条件**：逻辑会处理可能为空的集合、0、最大 / 最小值、首尾元素。
- **后置条件**：空集合、单元素、最大页、零值等边界场景有明确且正确的行为，不抛未捕获异常。
- **不变式**：分页 / 聚合 / 除法等运算对边界输入有定义（如除零被显式处理）。
- **验收标准**：
  - 空列表求平均 / 分页 → 返回空结果而非异常。
  - 除数为 0 / 整型溢出边界 → 被显式处理，不崩溃。

## 现场提问骨架

- "非法输入（缺字段、类型错、越界）是在哪一层拦的？会不会脏数据落库？"
- "空列表、除零、整型溢出这些边界场景，行为是确定的吗？"
