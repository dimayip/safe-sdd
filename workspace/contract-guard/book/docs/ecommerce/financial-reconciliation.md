# 15 · 资金与对账契约（financial-reconciliation）

> **适用领域**：`ecommerce`
> **适用场景**：支付、扣款、退款、优惠核销、账务记账、与第三方支付对账。
> **原则**：只否决"钱算错 / 重复入账 / 对不上账 / 凭空多扣少扣"，不绑定账务系统实现。

## 一句话理解

钱的事零容忍：每一笔变动总和必须守恒（有进必有出），回调重复不能重复入账，日终必须账实勾稽得上。

## 契约速览

| 契约 | 不变式（红线） |
| --- | --- |
| **FIN-1** 记账守恒、可勾稽 | 每笔交易各账户变动之和 = 0（复式记账） |
| **FIN-2** 支付回调幂等、不重复入账 | 同一支付回调多次到达，至多入账一次 |
| **FIN-3** 金额计算精确、无浮点误差 | 金额用定点 / 整数分计算，不丢精度 |

## 契约 FIN-1：记账守恒、可勾稽

- **前置条件**：存在多账户间的资金流转（用户、商户、平台、优惠）。
- **后置条件**：每笔交易记账后，相关账户的变动代数和为 0；任意时点可做账账 / 账实勾稽。
- **不变式**：`Σ各账户变动 = 0`；不存在凭空增减的资金。
- **验收标准**：日终对所有交易做勾稽 → 全部平衡，差异为 0 或可解释。

## 契约 FIN-2：支付回调幂等、不重复入账

- **前置条件**：第三方支付回调可能重复推送 / 乱序。
- **后置条件**：同一笔支付的多次回调至多入账一次；以支付流水号去重。
- **不变式**：同一支付流水号对应的入账副作用至多一次。
- **验收标准**：同一回调重复推送 5 次 → 入账金额只记一次，余额变动一次。

## 契约 FIN-3：金额计算精确、无浮点误差

- **前置条件**：涉及金额的乘除（折扣、分摊、税费）。
- **后置条件**：金额以整数分 / 定点小数计算，舍入规则明确一致；分摊后总和等于原值。
- **不变式**：不使用二进制浮点存储金额；分摊无累积误差。
- **验收标准**：100 元按 3 人分摊 → 三笔之和恰好 100 元，无 0.01 误差丢失。

## 现场提问骨架

- "重复提交 / 回调重投时，怎么保证只扣一次款、只入一次账？"
- "账务是复式记账吗？日终怎么对账、差异怎么追？"
- "金额是用 float 存的吗？折扣分摊会不会差一分钱？"
