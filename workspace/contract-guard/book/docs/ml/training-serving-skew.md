# 12 · 训练-服务一致性契约（training-serving-skew）

> **适用领域**：`ml-serving` `recommendation`
> **适用场景**：模型离线训练 + 在线推理两套链路，需要保证两者"看到的世界一致"。
> **原则**：只否决"训练/服务特征不一致 / 标签穿越 / 不可复现"，不绑定特征平台实现。

## 一句话理解

线上推理用的特征，必须和训练时算出来的是同一套口径；而且训练时绝不能偷看到"未来才知道的信息"。

## 契约速览

| 契约 | 不变式（红线） |
| --- | --- |
| **TSS-1** 训练/服务特征同源同口径 | 同一样本的训练特征与线上特征在容差内相等 |
| **TSS-2** 无未来信息泄漏（防穿越） | 特征只用样本时间点之前可得的信息 |
| **TSS-3** 训练可复现 | 同数据 + 同配置 + 同种子 → 同模型 |

## 契约 TSS-1：训练/服务特征同源同口径

- **前置条件**：离线特征与在线特征分别由不同链路计算。
- **后置条件**：同一样本在训练与推理时得到的特征值在容差内一致。
- **不变式**：特征定义（口径、单位、默认值、分桶）训练与服务侧一致。
- **验收标准**：抽样对比同一 key 的离线特征与在线特征 → 差异在容差内（skew 监控）。

## 契约 TSS-2：无未来信息泄漏（防标签穿越）

- **前置条件**：构造训练样本时关联了多张时间相关的表。
- **后置条件**：每个样本的特征仅使用该样本事件时间点之前可获得的信息。
- **不变式**：不存在用"未来"信息预测"过去"的特征（时间穿越）。
- **验收标准**：用 point-in-time 正确性校验 → 无晚于样本时间的信息进入特征。

## 契约 TSS-3：训练可复现

- **前置条件**：训练涉及随机性（初始化、采样、shuffle）。
- **后置条件**：固定数据版本 + 配置 + 随机种子，重训得到一致（或统计等价）的模型。
- **不变式**：训练输入与产物的版本可追溯绑定。
- **验收标准**：相同输入重跑两次 → 指标在可接受波动内一致，产物版本可回溯。

## 现场提问骨架

- "线上特征和训练特征是同一套口径吗？有没有 skew 监控？"
- "构造样本时怎么防标签穿越 / 未来信息泄漏？"
- "换个人 / 换台机器重训，结果可复现吗？"
