# 13 · 推理服务稳定性契约（inference-serving）

> **适用领域**：`ml-serving` `recommendation`
> **适用场景**：模型作为在线服务对外提供预测，对时延、稳定性、数值正确性有要求。
> **原则**：只否决"推理超时拖垮 / 输出 NaN / 漂移无感知 / 显存炸"，不绑定推理框架。

## 一句话理解

模型服务要在规定时间内出结果、输出不能是 NaN 或越界值、数据分布漂移了要能察觉、显存不能被打爆。

## 契约速览

| 契约 | 不变式（红线） |
| --- | --- |
| **INF-1** 推理时延有上界、可降级 | 推理耗时 ≤ 约定上界，超时走兜底 |
| **INF-2** 输出数值稳定 | 输出无 NaN / Inf，落在合法值域 |
| **INF-3** 输入分布漂移可感知 | 线上输入分布偏离训练分布时可被监控告警 |

## 契约 INF-1：推理时延有上界、可降级

- **前置条件**：推理服务有 SLA 时延要求。
- **后置条件**：单次推理耗时有上界；超时 / 过载时降级（小模型 / 缓存 / 兜底）而非堆积。
- **不变式**：推理路径上的资源（显存 / 批队列）占用有界。
- **验收标准**：注入慢请求 / 高并发 → P99 在上界内或走降级，不雪崩、不 OOM。

## 契约 INF-2：输出数值稳定

- **前置条件**：模型输出参与下游决策。
- **后置条件**：输出不含 NaN / Inf，落在业务合法值域（如概率 ∈ [0,1]）。
- **不变式**：任何输入都不产生非法数值输出。
- **验收标准**：注入极端 / 异常输入 → 输出仍合法或被显式拦截兜底，无 NaN 透传下游。

## 契约 INF-3：输入分布漂移可感知

- **前置条件**：线上数据分布可能随时间偏离训练分布。
- **后置条件**：关键输入特征的分布偏移被持续监控，超阈值告警。
- **不变式**：漂移不会在无感知的情况下持续劣化效果。
- **验收标准**：注入分布偏移 → 漂移监控触发告警。

## 现场提问骨架

- "推理超时怎么办？是堆积等死还是降级兜底？P99 有上界吗？"
- "模型可能输出 NaN 吗？输出值域有没有兜底校验？"
- "线上数据漂移了你们能发现吗？"
