# 分区容错与共识契约（partition-tolerance）

> 领域：分布式计算 / 多副本 / 集群协调。
> 原则：只否决"网络分区下脑裂双主 / CAP 取舍不明 / 时钟乱序导致因果倒置"，不绑定共识算法。

## 契约 PT-1：网络分区下的取舍被显式声明且不脑裂
- 前置条件：系统由多节点组成，可能发生网络分区。
- 后置条件：分区发生时，系统行为符合预先声明的 CAP 取舍（CP：拒绝写以保一致 / AP：可写但标记需收敛）；绝不出现两个分区各自接受冲突写且都自认为权威（脑裂）。
- 不变式：任意时刻对同一资源最多一个分区可执行权威写（基于多数派/租约）。
- 验收标准：
  - 用例：注入分区 → 少数派分区拒绝权威写或降级只读，不产生双主冲突写。
  - 用例：分区恢复 → 状态按声明策略收敛，冲突可检测并解决。

## 契约 PT-2：因果顺序不被时钟偏差破坏
- 前置条件：跨节点事件依赖先后关系，但物理时钟不可信。
- 后置条件：有因果依赖的事件用逻辑时钟/版本向量定序，不依赖墙上时钟先后判定因果。
- 不变式：若事件 A 因果先于 B，则系统判定的顺序中 A 不晚于 B。
- 验收标准：
  - 用例：节点间时钟回拨/偏移 → 因果序仍正确，不出现"结果先于原因"。

## 契约 PT-3：背压与流控，过载可控降级
- 前置条件：上游产生速率可能超过下游处理能力。
- 后置条件：下游过载时通过背压/限流向上游传导压力，而非无限堆积内存直至崩溃。
- 不变式：队列/缓冲有上界，超界触发拒绝或降级而非 OOM。
- 验收标准：
  - 用例：上游突增 10x → 下游限流/拒绝部分请求，进程不 OOM、不雪崩。
