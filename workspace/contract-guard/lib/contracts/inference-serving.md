# 推理服务稳定性契约（inference-serving）

> 领域：AI 训推 / 在线推理服务。
> 原则：只否决"推理超时拖垮主链路 / 数值异常输出脏结果 / 漂移无感知"，不绑定推理引擎。

## 契约 INF-1：推理有超时与降级
- 前置条件：在线请求依赖模型推理，推理耗时受批大小/显存/负载影响波动。
- 后置条件：推理设有超时上限；超时或服务不可用时降级到兜底（默认值/规则/缓存结果），不阻塞或拖垮主链路（依赖 FR-3）。
- 不变式：单请求推理耗时有上界，超界即降级。
- 验收标准：
  - 用例：注入推理延迟/GPU 打满 → 请求在超时内降级返回，主链路 P99 不被击穿。

## 契约 INF-2：输出数值稳定，异常可拦截
- 前置条件：模型输出可能出现 NaN / Inf / 越界概率 / 全零等异常。
- 后置条件：异常输出在返回前被检测并拦截/兜底，绝不把 NaN、越界分数等脏结果透传给下游。
- 不变式：对外输出的分数/概率落在声明值域内（如 [0,1]），无 NaN/Inf。
- 验收标准：
  - 用例：构造导致 NaN 的输入 → 被拦截并走兜底，下游不收到 NaN。

## 契约 INF-3：数据/概念漂移可检测
- 前置条件：线上输入分布会随时间偏离训练分布。
- 后置条件：对关键特征分布与模型输出分布做漂移监控，超阈值告警并可触发重训/降级流程。
- 不变式：输入/输出分布偏移超过阈值时一定产生可观测信号。
- 验收标准：
  - 用例：注入分布偏移 → 漂移监控在声明窗口内告警，不静默劣化。
