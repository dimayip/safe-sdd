# 接口兼容性契约（api-compatibility）

> 领域：大型软件工程 / 多团队协作 / 长生命周期系统。
> 原则：只否决"接口变更击穿存量调用方 / schema 演进不兼容"，不绑定具体协议（REST/gRPC/消息）。

## 契约 API-1：对外契约向后兼容
- 前置条件：接口/消息 schema 已有存量调用方，且仍在演进。
- 后置条件：在同一主版本内，变更只做兼容性扩展（加可选字段、加枚举值兜底），不删字段、不改语义、不收紧约束；破坏性变更必须升主版本并保留旧版本过渡期。
- 不变式：旧调用方在不修改的情况下，调用同主版本接口始终可正常工作。
- 验收标准：
  - 用例：用旧版本契约的客户端调用新版本服务 → 行为不破坏。
  - 用例：消费者契约测试（CDC）在变更后全部通过，破坏性变更被 CI 拦截。

## 契约 API-2：未知字段/枚举的前向兼容
- 前置条件：新版本可能下发调用方尚不认识的字段或枚举值。
- 后置条件：调用方对未知字段忽略而非报错，对未知枚举走声明的默认分支，不崩溃。
- 不变式：解析端对未预期输入有确定的容错行为。
- 验收标准：
  - 用例：注入未知字段/枚举 → 调用方正常处理并走默认分支。
