# 防滥用与安全边界契约（abuse-security）

> 领域：跨领域（对外 API / 电商 / 推荐 / 大型工程通用）。
> 原则：只否决"重放攻击 / 越权 / 刷量 / 敏感数据外泄"，不绑定鉴权与风控实现。

## 契约 SEC-1：防重放
- 前置条件：对外请求可能被截获并重复提交（尤其支付、领券、登录）。
- 后置条件：携带一次性 nonce + 时间戳/签名，过期或重复的请求被拒绝。
- 不变式：同一已消费的请求凭证不可被二次接受。
- 验收标准：
  - 用例：重放已成功的签名请求 → 被拒绝。

## 契约 SEC-2：鉴权与最小权限（防越权）
- 前置条件：资源归属特定主体，存在多租户/多用户。
- 后置条件：每次访问校验"主体是否有权操作该资源"，越权访问被拒绝；默认拒绝（白名单）而非默认允许。
- 不变式：主体只能访问其被授权的资源（水平 + 垂直越权均被阻断）。
- 验收标准：
  - 用例：A 用户带自己 token 访问 B 用户资源 → 被拒绝（防水平越权）。
  - 用例：普通用户访问管理接口 → 被拒绝（防垂直越权）。

## 契约 SEC-3：限流防刷
- 前置条件：接口可能被高频刷取（薅羊毛、爬虫、攻击）。
- 后置条件：对维度（用户/IP/设备）做频率限制，超限拒绝并记录，保护后端不被打垮。
- 不变式：单维度请求速率不超过声明阈值。
- 验收标准：
  - 用例：单用户超频请求 → 被限流，正常用户不受影响。
