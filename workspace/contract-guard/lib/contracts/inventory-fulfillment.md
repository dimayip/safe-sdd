# 库存与履约契约（inventory-fulfillment）

> 领域：电商交易 / 库存 / 订单履约。
> 原则：只否决"超卖 / 扣减与回滚不对称 / 履约链路中断无补偿"，不绑定锁与库存模型实现。

## 契约 INV-1：不超卖
- 前置条件：商品库存有限，高并发抢购/秒杀场景。
- 后置条件：成功下单总量永不超过可售库存；售罄后请求明确失败而非超卖（依赖 CONC-2）。
- 不变式：可用库存 ≥ 0 恒成立，无负库存。
- 验收标准：
  - 用例：库存 10，1000 并发下单 → 恰好 10 单成功，无负库存。

## 契约 INV-2：库存扣减与释放对称
- 前置条件：下单预占库存、支付后真扣、取消/超时未支付需释放。
- 后置条件：每一次预占都有对应的"确认扣减"或"释放归还"，不存在只占不放的泄漏，也不存在重复释放导致虚增。
- 不变式：预占量 = 已确认扣减量 + 待释放量；释放幂等（依赖 IDEM-1）。
- 验收标准：
  - 用例：下单未支付超时 → 预占库存被释放且仅释放一次。
  - 用例：重复触发取消 → 库存不被重复归还导致虚增。

## 契约 INV-3：履约可补偿（Saga 式最终完成或回滚）
- 前置条件：履约跨多步（扣库存→生成发货单→通知物流），任一步可能失败。
- 后置条件：履约要么最终全部完成，要么对已完成步骤执行补偿回滚，不停留在"扣了库存没发货且不退"的中间态（依赖 DC-1）。
- 不变式：每个正向步骤都有对应补偿动作，链路有终态。
- 验收标准：
  - 用例：发货环节失败 → 触发库存回补与订单状态回滚，资金按 FIN-2 处理，无残留。
