# 实时系统与 IoT 契约（realtime-iot）

> 领域：IoT / 实时控制 / 时序数据 / 边缘计算。
> 原则：只否决"时延不可控 / 设备离线无降级 / 时序错乱 / 指令重复执行致物理副作用"，不绑定具体协议（MQTT/CoAP）。

## 契约 RT-1：关键路径时延有硬上界
- 前置条件：存在对响应时延敏感的实时控制/反馈链路。
- 后置条件：关键操作的端到端时延有声明的硬上界（如 < 100ms）；超界视为失败并触发安全动作，而非无限等待。
- 不变式：实时路径耗时 ≤ 声明上界，超界即进入降级/安全态。
- 验收标准：
  - 用例：注入处理延迟 → 超界请求被判失败并执行兜底，不悬挂。
  - 用例：压测下 P99 时延 → 不超过声明上界。

## 契约 RT-2：设备离线/断连有确定行为
- 前置条件：设备网络不稳定，会频繁离线、重连。
- 后置条件：设备离线时系统有明确状态（last-will/标记离线），不把陈旧数据当实时数据；重连后状态可同步、不丢关键事件（依赖本地缓冲）。
- 不变式：被消费的数据带新鲜度标记，过期数据不被当作当前状态使用。
- 验收标准：
  - 用例：设备断连 → 在声明超时内被标记离线，不持续展示陈旧值为"在线最新"。
  - 用例：断连期间产生的关键事件 → 重连后补传，不丢失。

## 契约 RT-3：下行指令幂等且时序安全
- 前置条件：向设备下发会产生物理副作用的指令（开关、转动、投药），可能重发。
- 后置条件：指令携带序号/幂等键，设备对重复指令幂等（依赖 IDEM-1）；过期/乱序指令被设备拒绝，不执行旧指令覆盖新状态。
- 不变式：同一逻辑指令的物理动作至多执行一次；设备不执行比当前状态更旧的指令。
- 验收标准：
  - 用例：重发"开阀"指令 → 阀门动作仅一次。
  - 用例：旧指令晚于新指令到达 → 被设备按序号拒绝，不回退状态。

## 契约 RT-4：时序数据完整且时间基准统一
- 前置条件：多设备上报时序数据，各自本地时钟可能漂移。
- 后置条件：时序数据带统一时间基准（设备时间 + 服务端校正/逻辑时钟），乱序入库可纠正；采样缺口被标记而非静默插值伪造。
- 不变式：时序点时间戳口径一致；缺失数据可识别，不被伪造。
- 验收标准：
  - 用例：设备时钟漂移 → 入库时间被校正，跨设备时序可对齐。
  - 用例：采样中断 → 缺口被标记，不静默用插值冒充真实采样。
